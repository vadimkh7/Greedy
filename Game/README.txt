						GreedyAlgo

	Однажды программист решил разработать жадный алгоритм для решения задачи. Но на одном
алгоритме он остановиться не сумел.

	Игра представляет собою поле, на котором находятся следующие элементы:
- Стартовая точка игрока (M);
- Стена (#);
- «Фичи» разного номинала (3, 5, 7).

	У игрока есть два счетчика: число накопленных баллов и число оставшихся ходов. Игрок может
перемещаться по полю в четырех направлениях (<,^,>,v), расходуя при этом оставшиеся ходы.
Изначально у игрока в запасе 40 ходов. Как только игрок пересекается с объектом «Фича», игроку
прибавляется 3, 5 или 7 баллов и в два раза больше дополнительных ходов, старый объект фичи
удаляется и в случайной позиции (на свободной ячейке) появляется новый объект «Фича» со
случайным номиналом 3, 5 или 7. Игра заканчивается, как только у игрока не остается больше
ходов

	Пример уровня:

	#########
	# #     #
	#5# #3  #
	#   #   #
	# M  #  #
	# ##  # #
	#     7 #
	#########
